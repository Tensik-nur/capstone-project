# Руководство по разработке CyberShield Academy

## 🎯 Цели проекта

1. **Образование** - Предоставить структурированное обучение кибербезопасности
2. **Безопасность** - Следовать лучшим практикам безопасности веб-приложений
3. **Интерактивность** - Создать привлекательный и интерактивный опыт обучения
4. **Масштабируемость** - Архитектура для легкого расширения функциональности

## 🔒 Требования к безопасности

### Frontend Security

- ✅ Использовать Content Security Policy (CSP)
- ✅ Валидация всех пользовательских входов
- ✅ Защита от XSS (Cross-Site Scripting)
- ✅ Использование HTTPS при развертывании в продакшене
- ✅ Избегать встроенных скриптов (inline scripts)

### Backend Security (при добавлении)

- [ ] Осуществлять валидацию на серверной стороне
- [ ] Использовать CORS для ограничения доступа
- [ ] Хеширование паролей (bcrypt, argon2)
- [ ] Защита от SQL-инъекций (ORM, параметризованные запросы)
- [ ] Rate limiting для API
- [ ] HTTPS/TLS шифрование

### Data Protection

- [ ] Шифрование чувствительных данных
- [ ] Безопасное хранилище учетных данных
- [ ] Управление сессиями пользователей
- [ ] Регулярное резервное копирование

## 📁 Соглашения по структуре файлов

### CSS Структура (css/main.css)

- Переменные CSS в корне (`:root`)
- Reset стилей для всех элементов
- Классы и ID селекторы вместо тегов
- Медиа-запросы в конце каждого компонента
- Комментарии для больших блоков кода

### JavaScript Структура (js/main.js)

- Функции инициализации (initializeXxx)
- DOM-готовность (DOMContentLoaded)
- Обработчики событий в отдельных функциях
- Избегание глобальных переменных
- Комментарии для каждой функции

### HTML Структура

- Семантические теги (header, nav, main, footer, section)
- Правильная иерархия заголовков (h1 -> h2 -> h3)
- Доступность (alt для изображений, labels для форм)
- Относительные пути для файлов

## 🔄 Рабочий процесс разработки

### Добавление новой страницы

1. Создать папку в `pages/` если необходимо
2. Создать HTML файл с правильными ссылками на CSS и JS
3. Использовать существующий шаблон header и footer
4. Добавить ссылку в навигацию
5. Протестировать на разных размерах экранов

### Добавление новой функциональности в JavaScript

1. Создать новую функцию `initializeXxx()`
2. Добавить ее вызов в конец `DOMContentLoaded` слушателя
3. Комментировать код для читаемости
4. Тестировать в консоли браузера

### Обновление стилей

1. Перейти к соответствующему разделу в `css/main.css`
2. Обновить переменные CSS если нужно глобальное изменение
3. Добавить медиа-запросы для адаптивности
4. Протестировать на разных разрешениях

## 📊 Структура страницы

Все страницы должны следовать этому шаблону:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <!-- Meta теги -->
    <!-- Шрифты -->
    <!-- CSS -->
  </head>
  <body>
    <!-- Пиксельный фон -->
    <div class="pixel-grid"></div>

    <!-- Header с навигацией -->
    <header>...</header>

    <!-- Основное содержимое -->
    <main>
      <!-- Hero секция -->
      <section class="hero">...</section>

      <!-- Основной контент -->
      <section class="features">...</section>
    </main>

    <!-- Footer -->
    <footer>...</footer>

    <!-- Scripts -->
    <script src="path/js/main.js"></script>
  </body>
</html>
```
